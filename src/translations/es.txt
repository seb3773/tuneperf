# --- Parameter and command explanations dictionary ---
vm.swappiness|Controla la preferencia del kernel para intercambiar memoria anónima vs almacenar en caché archivos. Valores más bajos reducen el intercambio y la latencia, valores más altos aumentan el rendimiento bajo presión de memoria.
vm.vfs_cache_pressure|Controla la tendencia del kernel a recuperar objetos de directorio e inode de la caché. Valores más bajos mantienen los metadatos del sistema de archivos en caché, mejorando la responsividad para operaciones de archivo repetidas.
vm.dirty_background_ratio|Porcentaje de memoria del sistema que puede llenarse con páginas dirty (no escritas) antes de que el demonio de escritura en segundo plano comience a escribirlas en disco.
vm.dirty_ratio|Porcentaje de memoria del sistema que puede llenarse con páginas dirty antes de que cualquier proceso activo sea forzado a bloquear y escribir datos dirty en disco.
vm.dirty_background_bytes|Número de bytes dirty en los que el demonio de escritura en segundo plano comienza a escribir datos. (Mutuamente exclusivo con dirty_background_ratio).
vm.dirty_bytes|Bytes dirty en los que los procesos activos son forzados a bloquear y escribir datos. (Mutuamente exclusivo con dirty_ratio).
vm.dirty_expire_centisecs|Define cuánto tiempo pueden permanecer los datos dirty en memoria antes de ser considerados lo suficientemente viejos para ser escritos de nuevo en disco (en centésimas de segundo).
vm.dirty_writeback_centisecs|Define el intervalo en el que el demonio de escritura en segundo plano se ejecuta para verificar páginas dirty para escribir en disco (en centésimas de segundo).
vm.max_map_count|Número máximo de áreas de mapeo de memoria que un proceso puede tener. Crucial para aplicaciones pesadas (compiladores, bases de datos, juegos AAA vía Proton) para prevenir fallos de falta de memoria.
vm.compaction_proactiveness|Determina cuán proactivamente el kernel compacta la memoria para crear páginas libres contiguas. Valores más altos mantienen las hugepages disponibles pero pueden aumentar el overhead de CPU en segundo plano.
vm.watermark_boost_factor|Controla la agresividad del aumento de recuperación de memoria cuando se detecta fragmentación. Deshabilitar o bajar esto reduce micro-stutter bajo uso pesado de RAM.
vm.watermark_scale_factor|Define el tamaño de paso de marcas de agua de memoria libre. Aumentar esto hace que la recuperación de memoria se active antes, evitando picos de latencia de asignación directa.
vm.zone_reclaim_mode|Controla el reuso de memoria de nodo local en sistemas NUMA. Deshabilitar esto permite asignaciones de página desde otros nodos NUMA, evitando evicciones agresivas de caché local.
vm.page-cluster|Controla el número de páginas leídas del swap en un solo intento. Valores más bajos (como 0) reducen latencia para SSD/NVME, valores más grandes están optimizados para HDD mecánicos lentos.
vm.min_free_kbytes|Cantidad mínima de memoria libre (en KB) que el kernel preserva para evitar deadlock durante recuperación de memoria.
vm.dirtytime_expire_seconds|Tiempo en segundos después del cual metadatos dirty/time en caché deben sincronizarse con disco.
vm.extfrag_threshold|Umbral que indica cuando el kernel debería comenzar a compactar memoria en lugar de recuperar páginas debido a fragmentación externa.
vm.stat_interval|Frecuencia (en segundos) a la que el kernel actualiza estadísticas de memoria virtual, equilibrando overhead de rendimiento vs precisión.
vm.overcommit_memory|Define la política de overcommit de memoria del kernel. Modo 1 permite todas las asignaciones, mientras Modo 2 impone un límite estricto basado en swap y RAM.
vm.overcommit_ratio|Porcentaje de RAM física usado para calcular el límite estricto de asignación de memoria bajo modo overcommit 2.
vm.nr_hugepages|Número de hugepages persistentes pre-asignadas (usualmente 2MB) asignadas en RAM para virtualización o aceleración de base de datos.
kernel.sched_child_runs_first|Dice al planificador que ejecute primero los procesos hijo después de fork. Útil para shells y GUI interactivas para reducir latencia.
kernel.sched_autogroup_enabled|Agrupa tareas por sesión TTY para aislar automáticamente tareas de compilación/build de tareas GUI de escritorio, manteniendo la interfaz de escritorio fluida.
kernel.sched_cfs_bandwidth_slice_us|Establece el tamaño cuántico para límites de cuota CPU CFS. Valores más bajos reducen latencia de contenedor, valores más altos mejoran rendimiento.
kernel.sched_latency_ns|Período del planificador para ejecutar todas las tareas. Valores más bajos reducen latencia para tareas de escritorio/audio, valores más altos mejoran eficiencia de servidor/rendimiento.
kernel.sched_migration_cost_ns|Tiempo que el planificador considera una tarea 'cache hot' antes de migrarla a otro núcleo CPU. Previene ping-pong de tareas y thrashing de caché.
kernel.sched_min_granularity_ns|Duración mínima garantizada de ejecución de una tarea en CPU antes de ser pre-empted. Valores más bajos mejoran responsividad bajo contención.
kernel.sched_wakeup_granularity_ns|Controla cuán fácilmente tareas en despertar pueden pre-empt tareas actualmente en ejecución. Valores más bajos mejoran latencia de despertar pero pueden aumentar overhead de cambio de contexto.
kernel.numa_balancing|Habilita o deshabilita migración automática de tareas y páginas de memoria entre nodos NUMA. Deshabilitar esto elimina overhead de equilibrio en segundo plano en configuraciones de un solo socket.
kernel.sysrq|Controla qué operaciones pueden activarse mediante secuencias de teclas SysRq. Valor 1 habilita todas las funciones para recuperación, mientras valores más bajos las restringen por seguridad.
kernel.nmi_watchdog|Habilita el watchdog NMI para detectar bloqueos hardware. Deshabilitar esto reduce consumo de energía y ahorra interrupciones de timer de hardware.
kernel.panic|Número de segundos que el kernel espera después de un panic antes de reiniciar automáticamente. Valor 0 deshabilita reinicio automático, forzando al sistema a detenerse para debug.
kernel.panic_on_oops|Determina si un oops del kernel (crash menor) activa inmediatamente un panic (reinicio). Recomendado para servidores para evitar estados corruptos descontrolados.
fs.aio-max-nr|Número máximo de operaciones I/O asíncronas concurrentes permitidas a nivel de sistema. Crucial para bases de datos y aplicaciones I/O de alto rendimiento.
kernel.timer_migration|Permite al planificador migrar timers entre núcleos CPU para conservar energía en núcleos inactivos.
kernel.core_uses_pid|Agrega el ID de Proceso (PID) a nombres de archivo de core dump, facilitando el debug de múltiples procesos crash.
kernel.sched_nr_migrate|Controla el número de tareas migradas entre CPU en una sola ejecución del planificador para equilibrar overhead y responsividad.
kernel.shmmax|Tamaño máximo de un solo segmento de memoria compartida (en bytes) permitido.
kernel.shmall|Número total máximo de páginas de memoria compartida permitidas a nivel de sistema.
kernel.msgmax|Tamaño máximo (en bytes) de un solo mensaje en una cola de mensajes System V.
kernel.msgmnb|Tamaño máximo (en bytes) de una cola de mensajes System V.
net.core.somaxconn|Tamaño máximo de cola backlog para sockets TCP escuchando. Esencial aumentar para servidores de alta concurrencia para prevenir caídas de conexión.
net.core.netdev_max_backlog|Número máximo de paquetes en cola de entrada de red cuando CPU recibe paquetes más rápido que el driver de interfaz puede procesarlos.
net.core.rmem_max|Tamaño máximo de buffer de socket de recepción (en bytes). Crucial para redes de alto ancho de banda para sostener rendimiento máximo.
net.core.wmem_max|Tamaño máximo de buffer de socket de envío (en bytes). Crucial para redes de alto ancho de banda para sostener rendimiento máximo.
net.core.rmem_default|Tamaño predeterminado de buffer de socket de recepción (en bytes).
net.core.wmem_default|Tamaño predeterminado de buffer de socket de envío (en bytes).
net.core.optmem_max|Tamaño máximo de buffer auxiliar asignado por socket para datos de control (headers auxiliares).
net.ipv4.tcp_rmem|Tamaños mínimo, predeterminado y máximo de buffers de recepción para sockets TCP. El sistema auto-ajusta tamaños de buffer entre estos límites basado en presión de memoria.
net.ipv4.tcp_wmem|Tamaños mínimo, predeterminado y máximo de buffers de envío para sockets TCP. El sistema auto-ajusta tamaños de buffer entre estos límites basado en presión de memoria.
net.ipv4.tcp_max_syn_backlog|Número máximo de solicitudes SYN en cola. Ayuda a resistir ataques SYN flood y manejar grandes ráfagas de nuevas conexiones entrantes.
net.ipv4.tcp_slow_start_after_idle|Determina si la ventana de congestión TCP resetea al estado inicial después de que la conexión va inactiva. Deshabilitar mantiene alta velocidad para ráfagas de tráfico.
net.ipv4.tcp_tw_reuse|Permite al kernel reciclar en seguridad sockets TIME_WAIT para nuevas conexiones. Extremadamente beneficioso para servidores de alta concurrencia.
net.ipv4.tcp_fin_timeout|Tiempo de espera para un paquete FIN antes de cerrar un socket. Reducir esto libera sockets muertos más rápido, ahorrando memoria del sistema.
net.ipv4.tcp_keepalive_time|Cuánto tiempo debe estar inactiva una conexión TCP antes de enviar sondas keepalive.
net.ipv4.tcp_keepalive_intvl|Intervalo entre sondas keepalive individuales.
net.ipv4.tcp_keepalive_probes|Número máximo de sondas keepalive a enviar antes de abandonar la conexión.
net.ipv4.tcp_fastopen|Habilita TCP Fast Open, permitiendo intercambio de datos durante handshake inicial, reduciendo latencia de establecimiento de conexión.
net.ipv4.tcp_congestion_control|Selecciona el algoritmo de control de congestión TCP. BBR es recomendado para enlaces WAN de alta latencia, cubic/reno para uso general.
net.ipv4.tcp_notsent_lowat|Limita datos TCP no enviados en cola en los buffers. Minimiza bufferbloat y latencia en sockets locales.
net.ipv4.tcp_ecn|Habilita Notificación Explícita de Congestión (ECN) para mitigar pérdida de paquetes solicitando a routers throttling en lugar de drop de paquetes.
net.ipv4.tcp_sack|Habilita Reconocimiento Selectivo (SACK), permitiendo al receptor reconocer paquetes fuera de orden, evitando retransmisiones redundantes.
net.ipv4.tcp_dsack|Habilita SACK Duplicado (DSACK) para informar al remitente si un paquete fue retransmitido innecesariamente, mejorando precisión de control de congestión.
net.ipv4.tcp_fack|Habilita Reconocimiento Forward (FACK) para reducir retransmisiones innecesarias cuando múltiples paquetes se pierden en un vuelo.
net.ipv4.tcp_low_latency|Dice al stack TCP que priorice baja latencia sobre rendimiento máximo. Recomendado para gaming competitivo y comunicaciones en tiempo real.
net.ipv4.tcp_autocorking|Habilita coalescing automático de pequeñas escrituras en paquetes individuales. Deshabilitar reduce latencia para entregas de paquetes pequeñas inmediatas.
net.ipv4.ip_local_port_range|Rango de puertos usado para conexiones de cliente saliente. Ampliar esto previene agotamiento de puertos en servidores de alto tráfico.
net.ipv4.tcp_timestamps|Habilita Timestamps TCP para mejorar estimación de tiempo de ida y vuelta y proteger contra números de secuencia wrapped.
net.ipv4.tcp_retries2|Número máximo de veces que TCP intenta retransmitir un paquete no reconocido antes de abandonar la conexión.
net.ipv4.tcp_syn_retries|Número de paquetes SYN enviados antes de renunciar a un intento de conexión TCP saliente.
net.ipv4.tcp_synack_retries|Número de paquetes SYN/ACK enviados antes de abandonar un handshake de conexión TCP entrante.
net.ipv4.tcp_early_retrans|Habilita TCP Early Retransmit para recuperar pérdida de paquete más rápido bajo tamaños de ventana de congestión pequeños.
net.ipv4.tcp_mtu_probing|Detecta automáticamente tamaños de Path MTU (Unidad Máxima de Transmisión) cuando black holes ICMP bloquean entrega de paquetes.
net.ipv4.tcp_syncookies|Habilita cookies SYN para proteger el sistema contra ataques DDoS SYN flood verificando conexiones sin asignar memoria de socket.
net.ipv4.tcp_window_scaling|Habilita Escalado de Ventana TCP para soportar ventanas de recepción más grandes de 64KB, crítico para enlaces WAN de alta latencia.
