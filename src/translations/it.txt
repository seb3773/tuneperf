# --- Parameter and command explanations dictionary ---
vm.swappiness|Controlla la preferenza del kernel per lo swap di memoria anonima vs cache file. Valori più bassi riducono lo swap e la latenza, valori più alti aumentano il throughput sotto pressione memoria.
vm.vfs_cache_pressure|Controlla la tendenza del kernel a recuperare oggetti directory e inode dalla cache. Valori più bassi mantengono i metadati filesystem in cache, migliorando la responsività per operazioni file ripetute.
vm.dirty_background_ratio|Percentuale di memoria di sistema che può essere riempita con pagine dirty (non scritte) prima che il demone writeback in background inizi a scriverle su disco.
vm.dirty_ratio|Percentuale di memoria di sistema che può essere riempita con pagine dirty prima che qualsiasi processo attivo sia forzato a bloccare e scrivere dati dirty su disco.
vm.dirty_background_bytes|Numero di byte dirty a cui il demone writeback in background inizia a scrivere dati. (Mutuamente esclusivo con dirty_background_ratio).
vm.dirty_bytes|Numero di byte dirty a cui i processi attivi sono forzati a bloccare e scrivere dati. (Mutuamente esclusivo con dirty_ratio).
vm.dirty_expire_centisecs|Definisce per quanto tempo i dati dirty possono rimanere in memoria prima di essere considerati abbastanza vecchi da essere riscritti su disco (in centesimi di secondo).
vm.dirty_writeback_centisecs|Definisce l'intervallo a cui il demone writeback in background viene eseguito per controllare pagine dirty da scrivere su disco (in centesimi di secondo).
vm.max_map_count|Numero massimo di aree di mappamento memoria che un processo può avere. Cruciale per applicazioni pesanti (compilatori, database, giochi AAA via Proton) per prevenire crash out-of-memory.
vm.compaction_proactiveness|Determina quanto proattivamente il kernel compatta la memoria per creare pagine libere contigue. Valori più alti mantengono le hugepages disponibili ma possono aumentare l'overhead CPU in background.
vm.watermark_boost_factor|Controlla l'aggressività del boost di recupero memoria quando viene rilevata frammentazione. Disabilitare o abbassare questo riduce micro-stutter sotto uso RAM pesante.
vm.watermark_scale_factor|Definisce la dimensione del passo dei watermark memoria libera. Aumentare questo fa scattare il recupero memoria prima, evitando picchi latenza allocazione diretta.
vm.zone_reclaim_mode|Controlla il riutilizzo memoria nodo locale in sistemi NUMA. Disabilitare questo permette allocazioni pagina da altri nodi NUMA, evitando evict cache locali aggressivi.
vm.page-cluster|Controlla il numero di pagine lette dal swap in un solo tentativo. Valori più bassi (come 0) riducono latenza per SSD/NVME, valori più grandi sono ottimizzati per HDD meccanici lenti.
vm.min_free_kbytes|Quantità minima di memoria libera (in KB) che il kernel preserva per evitare deadlock durante recupero memoria.
vm.dirtytime_expire_seconds|Tempo in secondi dopo il quale metadati dirty/time cache devono essere sincronizzati su disco.
vm.extfrag_threshold|Soglia che indica quando il kernel dovrebbe iniziare a compattare memoria piuttosto che recuperare pagine a causa di frammentazione esterna.
vm.stat_interval|Frequenza (in secondi) a cui il kernel aggiorna statistiche memoria virtuale, bilanciando overhead prestazioni vs accuratezza.
vm.overcommit_memory|Definisce la politica overcommit memoria del kernel. Modalità 1 permette tutte le allocazioni, mentre Modalità 2 impone un limite rigoroso basato su swap e RAM.
vm.overcommit_ratio|Percentuale di RAM fisica usata per calcolare il limite rigoroso allocazione memoria sotto modalità overcommit 2.
vm.nr_hugepages|Numero di hugepages persistenti pre-allocate (solitamente 2MB) allocate in RAM per virtualizzazione o accelerazione database.
kernel.sched_child_runs_first|Dice allo scheduler di eseguire prima i processi figlio dopo fork. Utile per shell e GUI interattive per ridurre latenza.
kernel.sched_autogroup_enabled|Raggruppa task per sessione TTY per isolare automaticamente task compilazione/build da task GUI desktop, mantenendo l'interfaccia desktop fluida.
kernel.sched_cfs_bandwidth_slice_us|Imposta la dimensione quantum per limiti quota CPU CFS. Valori più bassi riducono latenza container, valori più alti migliorano throughput.
kernel.sched_latency_ns|Periodo scheduler per eseguire tutti i task. Valori più bassi riducono latenza per task desktop/audio, valori più alti migliorano efficienza server/throughput.
kernel.sched_migration_cost_ns|Tempo che lo scheduler considera un task 'cache hot' prima di migrarlo a un altro core CPU. Previene ping-pong task e thrashing cache.
kernel.sched_min_granularity_ns|Durata minima garantita di esecuzione di un task su CPU prima di essere pre-empted. Valori più bassi migliorano responsività sotto contesa.
kernel.sched_wakeup_granularity_ns|Controlla quanto facilmente task in risveglio possono pre-empt task attualmente in esecuzione. Valori più bassi migliorano latenza risveglio ma possono aumentare overhead switching contesto.
kernel.numa_balancing|Abilita o disabilita migrazione automatica task e pagine memoria tra nodi NUMA. Disabilitare questo rimuove overhead bilanciamento in background in configurazioni single-socket.
kernel.sysrq|Controlla quali operazioni possono essere attivate tramite sequenze chiave SysRq. Valore 1 abilita tutte le funzioni per recupero, mentre valori più bassi le limitano per sicurezza.
kernel.nmi_watchdog|Abilita watchdog NMI per rilevare lock hardware. Disabilitare questo riduce consumo energetico e risparmia interrupt timer hardware.
kernel.panic|Numero di secondi che il kernel attende dopo un panic prima di riavviare automaticamente. Valore 0 disabilita riavvio automatico, forzando il sistema a fermarsi per debug.
kernel.panic_on_oops|Determina se un oops kernel (crash minore) attiva immediatamente un panic (riavvio). Raccomandato per server per evitare stati corrotti runaway.
fs.aio-max-nr|Numero massimo di operazioni I/O asincrone concorrenti permesse a livello di sistema. Cruciale per database e applicazioni I/O alte prestazioni.
kernel.timer_migration|Permette allo scheduler di migrare timer tra core CPU per conservare energia su core idle.
kernel.core_uses_pid|Aggiunge l'ID Processo (PID) ai nomi file core dump, facilitando il debug di più processi crash.
kernel.sched_nr_migrate|Controlla il numero di task migrati tra CPU in una singola esecuzione scheduler per bilanciare overhead e responsività.
kernel.shmmax|Dimensione massima di un singolo segmento memoria condivisa (in byte) permessa.
kernel.shmall|Numero totale massimo di pagine memoria condivisa permesse a livello di sistema.
kernel.msgmax|Dimensione massima (in byte) di un singolo messaggio in una coda messaggi System V.
kernel.msgmnb|Dimensione massima (in byte) di un backlog coda messaggi System V.
